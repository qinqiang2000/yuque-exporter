# 飞书知识库文档拉取指南

> 本指南基于 `lark-cli` 的 `wiki` 和 `docs` 两个 Skill，介绍如何完整拉取飞书知识库中的文档内容。

---

## 一、身份选择（重要）

知识库操作**必须明确指定 `--as user`**，否则默认走 `bot` 身份，列出的是应用所属空间而非用户自己的空间。

```bash
# ✅ 正确：以用户身份操作
lark-cli wiki +space-list --as user

# ❌ 错误：bot 身份看不到用户个人知识库
lark-cli wiki +space-list
```

---

## 二、完整拉取流程（4 步）

### 第 1 步：列出所有可访问的知识空间

```bash
lark-cli wiki +space-list --as user --format json
```

返回结果中包含每个知识空间的：
- `space_id` — 空间唯一标识
- `name` — 空间名称
- `description` — 描述
- `space_type` — 类型（个人/共享）

---

### 第 2 步：列出知识空间下的文档节点

拿到 `space_id` 后，列出空间内所有文档（节点）：

```bash
# 列出根节点
lark-cli wiki +node-list --space-id <space_id> --as user --format json

# 列出某个父节点下的子节点（递归浏览）
lark-cli wiki +node-list --space-id <space_id> --parent-node-token <parent_token> --as user --format json
```

每个节点返回：
- `node_token` — 节点标识
- `obj_token` — 底层文档 token（用于拉取内容）
- `title` — 文档标题
- `has_child` — 是否有子节点

> **递归遍历**：如果 `has_child=true`，需继续用该节点的 `node_token` 作为 `--parent-node-token` 展开子节点。

---

### 第 3 步：通过节点信息获取文档 token

方法 A — 直接用 URL 获取节点信息：

```bash
lark-cli wiki +node-get --url "https://xxx.feishu.cn/wiki/<wiki_token>" --as user --format json
```

方法 B — 用 node_token 获取：

```bash
lark-cli wiki +node-get --node-token <node_token> --as user --format json
```

返回的 `obj_token` 和 `obj_type` 就是底层文档的真实 token 和类型。

---

### 第 4 步：拉取文档正文内容

拿到 `obj_token` 后，用 `docs +fetch` 读取文档内容：

```bash
# 读取完整文档内容（XML 格式，保留所有结构）
lark-cli docs +fetch --api-version v2 --doc <obj_token> --format xml

# 读取文档为 Markdown 格式
lark-cli docs +fetch --api-version v2 --doc <obj_token> --format markdown

# 只读取纯文本（不含格式标记）
lark-cli docs +fetch --api-version v2 --doc <obj_token> --format text

# 读取摘要/概览（不含全文，适合快速浏览）
lark-cli docs +fetch --api-version v2 --doc <obj_token> --scope summary
```

## 三、一键脚本：批量拉取整个知识空间

以下脚本可将一个知识空间的所有文档批量导出为 Markdown 文件：

```bash
#!/bin/bash
# 用法: ./pull_wiki.sh <space_id> [输出目录]
# 示例: ./pull_wiki.sh 7xxx9 ./output

SPACE_ID=$1
OUTPUT_DIR=${2:-"./wiki_export"}
mkdir -p "$OUTPUT_DIR"

echo "正在拉取知识空间: $SPACE_ID ..."

# 1. 列出所有根节点
NODES=$(lark-cli wiki +node-list --space-id "$SPACE_ID" --as user --format json)

# 2. 解析并递归导出（需配合 jq 或 Python 处理 JSON）
# 简化示例：逐个拉取根节点
echo "$NODES" | python3 -c "
import json, sys, subprocess, os

output_dir = '$OUTPUT_DIR'
nodes = json.load(sys.stdin).get('data', {}).get('items', [])

def pull_node(node, depth=0):
    title = node.get('title', 'untitled')
    obj_token = node.get('obj_token', '')
    node_token = node.get('node_token', '')
    safe_name = title.replace('/', '_').replace('\\\\', '_')

    if obj_token:
        print(f\"{'  '*depth}拉取: {title}\")
        out_file = os.path.join(output_dir, f'{safe_name}.md')
        result = subprocess.run(
            ['lark-cli', 'docs', '+fetch', '--api-version', 'v2',
             '--doc', obj_token, '--format', 'markdown'],
            capture_output=True, text=True
        )
        with open(out_file, 'w', encoding='utf-8') as f:
            f.write(result.stdout)
        print(f\"{'  '*depth}  -> {out_file}\")

    # 递归处理子节点
    if node.get('has_child'):
        child_result = subprocess.run(
            ['lark-cli', 'wiki', '+node-list',
             '--space-id', '$SPACE_ID',
             '--parent-node-token', node_token,
             '--as', 'user', '--format', 'json'],
            capture_output=True, text=True
        )
        children = json.load(sys.stdin if not child_result.stdout else open('/dev/stdin', 'r') if False else __import__('io').StringIO(child_result.stdout))
        # 简化：实际使用时需正确解析子节点 JSON

for node in nodes:
    pull_node(node)
"
```

## 四、常用场景速查

| 场景 | 命令 |
|------|------|
| 查看我有哪些知识空间 | `lark-cli wiki +space-list --as user` |
| 查看空间下的文档目录 | `lark-cli wiki +node-list --space-id <id> --as user` |
| 获取某个文档节点详情 | `lark-cli wiki +node-get --url <wiki_url> --as user` |
| 拉取文档全文（保留格式） | `lark-cli docs +fetch --api-version v2 --doc <token> --format xml` |
| 拉取文档为 Markdown | `lark-cli docs +fetch --api-version v2 --doc <token> --format markdown` |
| 只看文档摘要 | `lark-cli docs +fetch --api-version v2 --doc <token> --scope summary` |
| 读取文档中的图片 | `lark-cli docs +media-download --doc <token> --media-id <media_id>` |
| 复制节点到其他空间 | `lark-cli wiki +node-copy --node-token <token> --target-space-id <id>` |
| 移动节点 | `lark-cli wiki +move --node-token <token> --target-space-id <id>` |

## 五、注意事项

### 1. 权限要求

| 操作 | 所需 scope |
|------|-----------|
| 列出知识空间 | `wiki:space:retrieve` |
| 读取节点信息 | `wiki:node:read` |
| 读取节点列表 | `wiki:node:retrieve` |
| 读取文档内容 | 需要文档的阅读权限 |

### 2. 嵌套资源处理

文档中可能嵌入了其他资源，拉取后需注意识别：

| 嵌入类型 | 识别标记 | 处理方式 |
|---------|---------|---------|
| 电子表格 | `<sheet token="..." sheet-id="...">` | 用 `lark-sheets` 读取 |
| 多维表格 | `<bitable token="..." table-id="...">` | 用 `lark-base` 读取 |
| 画板 | `<whiteboard>` | 用 `lark-cli docs +media-download --type whiteboard` |
| 同步块 | `<synced_reference src-token="...">` | 用 `docs +fetch` 读取引用源文档 |

### 3. Bot 身份限制

- `--as bot` 只能看到**应用被授权访问**的知识空间，看不到用户个人空间
- 用 bot 身份**不能**添加部门类型的成员到知识空间
- 推荐始终使用 `--as user`，除非明确需要应用视角

### 4. 分页处理

节点列表默认只返回第一页，数据量大时需翻页：

```bash
# 自动翻页获取全部节点
lark-cli wiki +node-list --space-id <id> --as user --page-all --format json
```

## 六、与 lark-doc 的关系

```
知识库（Wiki）                文档（Docs）
┌─────────────┐            ┌─────────────────┐
│ space       │            │ document content │
│  └─ node ───┼──obj_token─┼──> +fetch 读正文 │
│     └─ child │            │   +update 改内容 │
└─────────────┘            └─────────────────┘
```

- **Wiki** 负责知识空间的**结构管理**（空间、节点、成员、层级）
- **Docs** 负责文档的**内容读写**（正文、格式、媒体）
- 拉取知识库文档 = Wiki 定位节点 → Docs 读取内容
